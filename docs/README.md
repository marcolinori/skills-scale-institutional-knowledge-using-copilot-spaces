# OctoAcme Project Management Processes

Welcome to the OctoAcme Project Management documentation hub. This directory contains comprehensive guides for managing projects using the OctoAcme approach—a framework designed to deliver customer value through iterative, well-coordinated, and transparent project execution.

## Overview

OctoAcme is built on five core principles that guide all project work:

- **Customer-first**: Prioritize customer value and usability in every decision
- **Iterative delivery**: Deliver small, testable increments to reduce risk and gather feedback early
- **Clear ownership**: Each project has named Project Manager (PM) and Product Lead with defined responsibilities
- **Data-informed decisions**: Measure impact and iterate based on evidence, not assumptions
- **Psychological safety**: Encourage feedback, learning, and innovation by creating safe spaces for candid discussion

These principles are embedded throughout the project lifecycle and reflected in every process document in this folder.

## How OctoAcme Works

OctoAcme follows a five-phase project lifecycle that ensures alignment, quality, and continuous learning:

1. **Initiation** — Validate business need, align stakeholders, and secure approval to move forward
2. **Planning** — Break work into shippable increments, identify dependencies, and create a delivery roadmap
3. **Execution** — Build and test incrementally, maintain team rhythm, and escalate blockers proactively
4. **Release** — Deploy to production with reduced risk, verification, and clear rollback procedures
5. **Retrospective** — Capture learnings and convert them into actionable improvements

Throughout all phases, the team maintains clear communication, proactively manages risks, and adheres to quality standards that ensure delivery excellence.

### 1. **Initiation** — Define & Validate
When a new project idea or feature proposal is ready to be explored, follow these steps:
- **[Project Initiation Guide](octoacme-project-initiation.md)** — Validate business need, confirm stakeholder alignment, and decide go/no-go for planning. Includes the Project One-pager template and initiation checklist.

### 2. **Planning** — Scope & Prepare
Once approved, turn the initiative into an actionable plan:
- **[Project Planning](octoacme-project-planning.md)** — Break work into shippable increments, identify dependencies, estimate scope, and create a release plan. Includes backlog item template, Definition of Done, and planning checklist.

### 3. **Execution** — Build & Iterate
Execute day-to-day delivery with team rhythm and quality standards:
- **[Execution & Tracking](octoacme-execution-and-tracking.md)** — Manage standups and syncs, use project boards effectively, run automated quality checks, track metrics, and escalate blockers through defined levels. Includes execution checklist and PR workflow standards.

### 4. **Release** — Deploy & Verify
Prepare for and execute production releases with confidence:
- **[Release & Deployment Guide](octoacme-release-and-deployment.md)** — Standardize release types, prepare pre-release requirements, execute deployment with verification, and maintain rollback procedures. Includes deployment checklist, incident playbook, and release notes template.

### 5. **Retrospective** — Learn & Improve
After each sprint, release, or milestone, capture learnings:
- **[Retrospective & Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md)** — Run retrospectives, track action items, and measure the impact of improvements. Includes retrospective structure, action item template, and continuous improvement culture guidance.

## How to Use These Docs

- **Keep artifacts updated in your project repository** — Project Charter, One-pager, backlog, risk register, and status documents should live in your project repo (typically in `docs/` or `.copilot/`)
- **Reference the appropriate phase document** — Use the lifecycle section above to find the guidance relevant to your current project phase
- **Adapt to your context** — These docs provide structure and best practices, but adapt them to your team's constraints, technical environment, and organizational needs
- **Add these docs to Copilot Spaces** — If using GitHub Copilot Spaces, add these documents as context to get role-specific guidance and process-aware assistance
- **Iterate on these processes** — Use retrospectives to identify process improvements and update these docs collaboratively as your team learns

## Key Metrics & Success Indicators

OctoAcme projects track:
- **Velocity and burndown** — Measure delivery pace and adjust capacity planning
- **Success metrics** — Progress toward objectives defined in the Project One-pager
- **Quality signals** — Test coverage, security scan results, error rates, deployment frequency
- **Team health** — Retrospective action items completed, psychological safety scores, communication clarity
